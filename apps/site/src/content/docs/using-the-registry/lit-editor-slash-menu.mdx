`lit-editor-slash-menu` is a `/`-triggered slash command menu for inserting nodes.

## Install

```bash
npx shadcn@latest add https://qti-editor.citolab.nl/r/lit-editor-slash-menu.json
```

The registry command installs the component source and its package dependencies into your app.

## Usage

```ts
import '@/components/slash-menu';

const menu = document.querySelector('lit-editor-slash-menu');
menu.editor = editor;
```

## How it works

Typing `/` at the start of an empty paragraph triggers the menu. A floating panel appears anchored to the cursor position and lists available commands. As the user continues typing, the list is filtered to matching items. Pressing `Enter` or clicking an item runs the associated command and removes the `/` trigger character. Pressing `Escape` or clicking outside the menu dismisses it without inserting anything.

The available items are derived from the registered interaction descriptors and basic block types (headings, lists, blockquote). Only commands valid at the current cursor position are shown; items requiring schema nodes that are not registered are excluded.

## Positioning

Place the slash menu as a sibling of your editor mount element in the DOM so that its absolute positioning stays inside the editor's scroll container. In the Angular shell example:

```html
<div class="editor-stage">
  <div #mount class="editor-mount"></div>
  <lit-editor-slash-menu #slashMenu class="editor-slash-menu"></lit-editor-slash-menu>
</div>
```
