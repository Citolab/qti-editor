`qti-choice-attributes-editor` is the standalone choice-layout editor used by `qti-attributes-panel`.

## Install

```bash
npx shadcn@latest add https://qti-editor.citolab.nl/r/qti-choice-attributes-editor.json
```

The registry command installs the component source and its package dependencies into your app.

## Usage

This component is usually installed indirectly through `qti-attributes-panel`, but you can also host it yourself.

```ts
import '@/components/choice-attributes-editor';

const editor = document.querySelector('qti-choice-attributes-editor');
editor.activeNode = activeNodeDetail;
editor.addEventListener('qti:attributes:patch', event => {
  const detail = (event as CustomEvent).detail;
  console.log(detail.pos, detail.attrs);
});
```

It expects an `activeNode` compatible with `AttributesNodeDetail` from `@qti-editor/prosemirror-attributes-ui` and emits `qti:attributes:patch` events when the author changes the choice layout classes.

## How it works

The editor surfaces layout presets for `qti-choice-interaction` — options that control how choices are visually arranged. Selecting a preset applies a CSS class string to the interaction node's `class` attribute. The component emits a `qti:attributes:patch` event with the updated `class` value and the node's document position:

```ts
// Event detail shape:
{
  pos: number;       // ProseMirror node position
  attrs: {
    class: string;   // updated CSS class string
  };
}
```

The parent (`qti-attributes-panel` or your own host) applies this patch as a ProseMirror attribute transaction.

## Layout presets

Presets control whether choices are stacked vertically (the default) or arranged in columns or inline rows. The exact preset options are defined in the copied component source and can be modified after installation.
