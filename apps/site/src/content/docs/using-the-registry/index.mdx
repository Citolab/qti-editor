The registry is an **optional** Lit-based UI starter. The [`qti-prosemirror-item`](https://github.com/Citolab/qti-editor/tree/main/apps/qti-prosemirror-item) boilerplate doesn't use it — it builds its own Insert menu with `prosemirror-menu` and its own attributes panel with a small local plugin. Use the registry when you want pre-built Lit components instead of authoring those pieces yourself.

The registry is delivered through the [shadcn/ui-compatible](https://ui.shadcn.com) format. QTI Editor has no dependency on shadcn/ui or its component library; the shadcn registry format is used purely as a delivery mechanism for copying source files into your project. Each `pnpm dlx shadcn@latest add` command downloads the component source into your codebase, installs any npm dependencies it needs, and leaves you with plain files you own and can modify freely.

Registry repo: `citolab/qti-editor`

## Prerequisites

These components are Lit web components styled with Tailwind CSS and DaisyUI. Your project needs:

```bash
pnpm add lit tailwindcss daisyui
```

If your framework is not auto-detected by the shadcn CLI, add some manual config first:

- `components.json`
- a `@/*` path alias in `tsconfig.json`
- a minimal `src/lib/utils.ts`

## Installing a component

Use the shadcn CLI with the GitHub item address:

```bash
pnpm dlx shadcn@latest add citolab/qti-editor/qti-attributes-panel
```

This copies the component source into your project and installs its npm dependencies. You can then modify the file freely.

After installation, import the copied files from your own app source tree.

## Component catalogue

- [QTI Attributes Panel](/docs/using-the-registry/qti-attributes-panel/) — schema-aware attribute editor for the current node selection
- [QTI Interaction Insert Menu](/docs/using-the-registry/qti-interaction-insert-menu/) — dropdown for inserting supported QTI interactions

Supporting registry items are pulled in automatically when needed:

- `qti-choice-attributes-editor`
- `qti-text-entry-attributes-editor`
- `qti-extended-text-attributes-editor`

## Notes

- Each component is a Lit custom element, which makes the registry usable in any HTML context.
- Assign object values such as the editor instance through JavaScript rather than HTML attributes.
- When the shadcn CLI installs a component, it copies the source files into your app and resolves internal imports through your `components.json` aliases. Make sure those aliases point at the directories you want to own.
- Components with `registryDependencies`, such as `qti-attributes-panel`, will automatically pull in their sub-components in a single CLI command.
- The copied registry components are optional. The stable integration surface is the public package API plus your own app-level editor assembly — see the [TypeScript Integration](/docs/frameworks/vanilla/) guide.
- The repository's root `registry.json` is the source of truth for the GitHub registry.
