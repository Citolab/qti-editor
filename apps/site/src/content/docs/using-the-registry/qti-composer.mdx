`qti-composer` displays the composed QTI XML output for the current item. The component reads metadata from the shared item context and recomposes from the live ProseKit editor instance.

This component lives in the demo app shell in this repository. It is not part of the GitHub registry, so there is no `shadcn add` command for it.

## Usage

Assign the editor instance after mount:

```ts
import './components/composer';

const composer = document.querySelector('qti-composer');
composer.editor = editor;
composer.liveComposeEnabled = true;
```

In a full editor shell, pair this with `qti-composer-metadata-form` and provide item metadata through the shared item context, as the demo app does.

## How it works

When `liveComposeEnabled` is `true`, the composer subscribes to content-change events from the editor and recomposes the full QTI `<assessmentItem>` XML on every document change. The composed output is shown in a read-only code view.

The composition combines:
- the current editor document (serialized to QTI item-body XML)
- item metadata — `identifier`, `title`, and `lang` — read from the item context or set directly on the element

The result is a valid QTI 3.0 `<assessmentItem>` structure. This is the same output that `@citolab/prose-qti/package-builder` packages into a ZIP — the composer just surfaces it live for inspection.

## Disabling live compose

Set `liveComposeEnabled = false` to pause recomposition. This is useful when the editor is in a loading or import state and you want to suppress intermediate output until the document is stable.

composer.liveComposeEnabled = false; // pause during import
await loadDocument(composer.editor);
composer.liveComposeEnabled = true;  // resume
