`qti-extended-text-attributes-editor` is the standalone extended-text editor used by `qti-attributes-panel`.

## Install

```bash
npx shadcn@latest add https://qti-editor.citolab.nl/r/qti-extended-text-attributes-editor.json
```

The registry command installs the component source and its package dependencies into your app.

## Usage

This component is usually installed indirectly through `qti-attributes-panel`, but you can also host it yourself.

```ts
import '@/components/extended-text-attributes-editor';

const editor = document.querySelector('qti-extended-text-attributes-editor');
editor.activeNode = activeNodeDetail;
editor.addEventListener('qti:attributes:patch', event => {
  const detail = (event as CustomEvent).detail;
  console.log(detail.pos, detail.attrs);
});
```

It emits `qti:attributes:patch` events for height presets and rubric/model-answer updates.

## How it works

The editor exposes two authoring controls for `qti-extended-text-interaction`:

| Control | What it controls |
|---|---|
| Height preset | Visual row height of the response area |
| Rubric / model answer | Scoring guide text stored on the node |

Height presets set a CSS class on the node that the `@qti-components/extended-text-interaction` rendering uses to size the response area. The rubric field is a free-text area for authors to record a model answer or scoring notes — it is stored as a node attribute and included in QTI export.

Each change emits a `qti:attributes:patch` event with `pos` and `attrs`, following the same pattern as the other attribute editors:

```ts
editor.addEventListener('qti:attributes:patch', (e) => {
  const { pos, attrs } = (e as CustomEvent).detail;
  // apply attrs at pos as a ProseMirror transaction
});
```
