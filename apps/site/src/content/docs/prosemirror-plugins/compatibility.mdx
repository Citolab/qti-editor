The compatibility helpers provide a versioned migration pipeline for editor documents. Use them when you need to upgrade stored JSON or HTML content across schema versions without dropping unknown content.

## Install

```bash
pnpm add @citolab/prose-extensions
```

## Core APIs

- `createMigrationRegistry()` - build a migration registry for a document type
- `migrateDocument()` - run a migration chain directly
- `migrateJsonDocument()` - migrate raw ProseMirror JSON
- `migrateHtmlFragment()` - migrate HTML fragments
- `readPersistedDoc()` - read stored JSON plus migration metadata
- `buildCompatibilityReport()` - summarize migration results

## Usage

```ts
import { createMigrationRegistry, JSON_MIGRATION_STEPS } from '@citolab/prose-extensions/compatibility';

const registry = createMigrationRegistry({
  targetVersion: 6,
  steps: JSON_MIGRATION_STEPS,
});
```

The compatibility chapter is not a runtime plugin. It is the migration layer that keeps stored documents readable as the schema evolves.
