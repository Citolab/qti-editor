`local-storage-doc-persistence-extension` stores the current editor document in `localStorage` and restores it on load. It is useful for autosave-style authoring flows and local drafts.

## Install

```bash
pnpm add @citolab/prose-extensions
```

## Usage

```ts
import {
  createLocalStorageDocPersistencePlugin,
  readPersistedStateFromLocalStorage,
} from '@citolab/prose-extensions/local-storage-doc-persistence-extension';

const storageKey = 'qti-editor-doc';
const persisted = readPersistedStateFromLocalStorage(storageKey);

const plugins = [createLocalStorageDocPersistencePlugin({ storageKey })];
```

Use `readPersistedStateFromLocalStorage()` before editor creation when you want to hydrate from a saved draft.

### ProseKit

If your app is built on ProseKit, use the extension wrapper instead. It requires the `prosekit` peer dependency:

```ts
import { defineLocalStorageDocPersistenceExtension } from '@citolab/prose-extensions/prosekit-extensions';

const persistenceExtension = defineLocalStorageDocPersistenceExtension({
  storageKey,
});
```

## When to use

Use this when you want local persistence without wiring your own save loop.
