QTI Editor ships its interactions as focused subpath exports from `@qti-editor/interactions`. Each subpath adds the schema, commands, serialization, web components, and any runtime ProseMirror plugins needed for one QTI interaction type.

These packages are closely coupled to our other open source library, [`qti-components`](https://github.com/Citolab/qti-components). In practice, QTI Editor is responsible for authoring behavior and document structure, while `qti-components` supplies the underlying QTI web components that render and encapsulate the interaction behavior itself. That split keeps the integration framework agnostic while letting each interaction package stay small and explicit.

When you install the umbrella package, the matching `@qti-components/*` dependencies are pulled in automatically through the individual subpaths. You typically import only the interactions your product needs.

## Available interactions

| Interaction | What it does | Package |
|---|---|---|
| [Associate Interaction](/docs/qti-interactions/associate/) | Associating options from a shared set according to configured matching rules | `@qti-editor/interactions/associate` |
| [Choice Interaction](/docs/qti-interactions/choice/) | Single- or multiple-choice questions with a prompt and selectable choices | `@qti-editor/interactions/choice` |
| [Gap Match Interaction](/docs/qti-interactions/gap-match/) | Dragging or placing predefined gap texts into matching gaps in a passage | `@qti-editor/interactions/gap-match` |
| [Hottext Interaction](/docs/qti-interactions/hottext/) | Marking words or phrases inside a text passage as selectable response options | `@qti-editor/interactions/hottext` |
| [Inline Choice Interaction](/docs/qti-interactions/inline-choice/) | Dropdown choices embedded directly in a sentence | `@qti-editor/interactions/inline-choice` |
| [Text Entry Interaction](/docs/qti-interactions/text-entry/) | Short free-text answers entered inline | `@qti-editor/interactions/text-entry` |
| [Extended Text Interaction](/docs/qti-interactions/extended-text/) | Longer free-text answers in a multi-line response area | `@qti-editor/interactions/extended-text` |
| [Match Interaction](/docs/qti-interactions/match/) | Matching items across two sets | `@qti-editor/interactions/match` |
| [Order Interaction](/docs/qti-interactions/order/) | Ordering a list of choices into the correct sequence | `@qti-editor/interactions/order` |
| [Select Point Interaction](/docs/qti-interactions/select-point/) | Clicking a coordinate or hotspot on an image | `@qti-editor/interactions/select-point` |

## Installing interactions

Install the umbrella package once, then import only the subpaths you need:

```bash
pnpm add @qti-editor/interactions
```

Each interaction subpath exports [ProseMirror](https://prosemirror.net/) commands, schema pieces, and web components. Interactions that need runtime behavior (like click-to-mark-correct-response or wrapping selected text as hottext) include their ProseMirror plugins in the subpath.

**Each interaction package provides:**

- **Pure ProseMirror plugins** - Plugin factory functions that return standard ProseMirror `Plugin` objects
- **ProseMirror commands** - Standard command functions for inserting and manipulating interactions
- **Node specs** - ProseMirror schema definitions
- **Web components** - Lit-based UI elements for rendering
- **Descriptor metadata** - Enough information to assemble your own ProseKit extension layer

**Pure ProseMirror setup:**

Use `listInteractionDescriptors()` for node specs and commands, and `listInteractionPluginFactories()` for the runtime plugin factory functions:

```typescript
import { listInteractionDescriptors, listInteractionPluginFactories } from '@qti-editor/core/interactions/composer';

const descriptors = listInteractionDescriptors();
const plugins = listInteractionPluginFactories().map(factory => factory());
```

**ProseKit setup:**

For external apps, build a small local ProseKit helper around the descriptor registry:

```typescript
import { listInteractionDescriptors, listInteractionPluginFactories } from '@qti-editor/core/interactions/composer';
import { registerQtiInteractionElements } from '@qti-editor/prosekit-integration';
import { defineKeymap, defineNodeSpec, definePlugin, union, type Extension } from 'prosekit/core';

import type { Command } from 'prosekit/pm/state';

export function defineQtiInteractionsExtension() {
  registerQtiInteractionElements();
  const descriptors = listInteractionDescriptors();
  const seenSpecs = new Set<string>();
  const nodeSpecExtensions: Extension[] = [];

  for (const descriptor of descriptors) {
    for (const { name, spec } of descriptor.nodeSpecs) {
      if (seenSpecs.has(name)) continue;
      seenSpecs.add(name);
      nodeSpecExtensions.push(defineNodeSpec({ name, ...spec }));
    }
  }

  const keymap: Record<string, Command> = {};
  const enterCommands = descriptors
    .map(descriptor => descriptor.enterCommand)
    .filter((command): command is Command => command != null);

  if (enterCommands.length > 0) {
    keymap['Enter'] = (state, dispatch, view) =>
      enterCommands.some(command => command(state, dispatch, view));
  }

  for (const descriptor of descriptors) {
    if (descriptor.insertCommand && descriptor.keyboardShortcut) {
      keymap[descriptor.keyboardShortcut] = descriptor.insertCommand;
    }
  }

  const pluginExtensions = listInteractionPluginFactories().map(pluginFactory =>
    definePlugin(pluginFactory),
  );

  return union(...nodeSpecExtensions, defineKeymap(keymap), ...pluginExtensions);
}
```

## Inserting an interaction

In the editor UI, use the **Interactions** toolbar button to open the insert menu. Only interactions whose schema nodes are registered in your editor extension are listed. Each item is disabled when the cursor is in a position where that node cannot be inserted.

Interactions can also be inserted programmatically using the command functions exported from each package:

```ts
import { insertChoiceInteraction } from '@qti-editor/interactions/choice';
import { insertHottextInteraction } from '@qti-editor/interactions/hottext';
import { insertTextEntryInteraction } from '@qti-editor/interactions/text-entry';
import { insertInlineChoiceInteraction } from '@qti-editor/interactions/inline-choice';
import { insertSelectPointInteraction } from '@qti-editor/interactions/select-point';

// Call as a command:
insertChoiceInteraction(view.state, view.dispatch, view);
```

## Attributes panel

When the cursor is inside any interaction node, the **Attributes** panel shows the schema-defined attributes for that node and its ancestors. Changes are applied live to the editor document.

Attributes vary by interaction type but commonly include:

| Attribute | Description |
|---|---|
| `responseIdentifier` | Links the interaction to a response declaration |
| `maxChoices` | Maximum number of selectable options (choice interaction) |
| `expectedLength` | Character limit hint (text entry / extended text) |
| `caseSensitive` | Whether the response match is case-sensitive (text entry) |

The [framework guides](/docs/frameworks/react/) and [framework guides](/docs/frameworks/angular/) show complete integration examples.
