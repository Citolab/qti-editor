QTI Editor ships its interactions as focused subpath exports. Each interaction has its own package subpath and a matching `register` entrypoint, and the smallest useful integration is usually to import and register one interaction at a time.

These packages are closely coupled to our other open source library, [`qti-components`](https://github.com/Citolab/qti-components). In practice, QTI Editor is responsible for authoring behavior and document structure, while `qti-components` supplies the underlying QTI web components that render and encapsulate the interaction behavior itself. That split keeps the integration framework agnostic while letting each interaction package stay small and explicit.

When you install `@citolab/prose-qti`, the matching `@qti-components/*` dependencies are pulled in automatically. You typically import only the interactions your product needs.

## Available interactions

| Interaction | What it does | Package |
|---|---|---|
| [Associate Interaction](/docs/qti-interactions/associate/) | Associating options from a shared set according to configured matching rules | `@citolab/prose-qti/components/associate` |
| [Choice Interaction](/docs/qti-interactions/choice/) | Single- or multiple-choice questions with a prompt and selectable choices | `@citolab/prose-qti/components/choice` |
| [Gap Match Interaction](/docs/qti-interactions/gap-match/) | Dragging or placing predefined gap texts into matching gaps in a passage | `@citolab/prose-qti/components/gap-match` |
| [Hottext Interaction](/docs/qti-interactions/hottext/) | Marking words or phrases inside a text passage as selectable response options | `@citolab/prose-qti/components/hottext` |
| [Inline Choice Interaction](/docs/qti-interactions/inline-choice/) | Dropdown choices embedded directly in a sentence | `@citolab/prose-qti/components/inline-choice` |
| [Text Entry Interaction](/docs/qti-interactions/text-entry/) | Short free-text answers entered inline | `@citolab/prose-qti/components/text-entry` |
| [Extended Text Interaction](/docs/qti-interactions/extended-text/) | Longer free-text answers in a multi-line response area | `@citolab/prose-qti/components/extended-text` |
| [Match Interaction](/docs/qti-interactions/match/) | Matching items across two sets | `@citolab/prose-qti/components/match` |
| [Order Interaction](/docs/qti-interactions/order/) | Ordering a list of choices into the correct sequence | `@citolab/prose-qti/components/order` |
| [Select Point Interaction](/docs/qti-interactions/select-point/) | Clicking a coordinate or hotspot on an image | `@citolab/prose-qti/components/select-point` |

## Installing interactions

Install `@citolab/prose-qti`, then import the interaction subpath and its `register` module:

```bash
pnpm add @citolab/prose-qti
```

Each interaction subpath exports [ProseMirror](https://prosemirror.net/) commands, schema pieces, and web components. Interactions that need runtime behavior (like click-to-mark-correct-response or wrapping selected text as hottext) include their ProseMirror plugins in the subpath.

**Each interaction package provides:**

- **Pure ProseMirror plugins** - Plugin factory functions that return standard ProseMirror `Plugin` objects
- **ProseMirror commands** - Standard command functions for inserting and manipulating interactions
- **Node specs** - ProseMirror schema definitions
- **Web components** - Lit-based UI elements for rendering
- **Descriptor metadata** - Enough information to assemble your own ProseKit extension layer

**Pure ProseMirror setup:**

Use `listInteractionDescriptors()` for node specs and commands, and `listInteractionPluginFactories()` for the runtime plugin factory functions:

```typescript
import { listInteractionDescriptors, listInteractionPluginFactories } from '@citolab/prose-qti/core/interactions/composer';

const descriptors = listInteractionDescriptors();
const plugins = listInteractionPluginFactories().map(factory => factory());
```

**ProseKit setup:**

`defineQtiInteractionsExtension` is a package export — import it directly:

```typescript
import '@citolab/prose-qti/components/choice/register';
import { defineQtiInteractionsExtension } from '@citolab/prose-qti/integration/interactions/prosekit';
```

Custom elements are registered through side-effect imports, not a runtime helper. Import `@citolab/prose-qti/components/<name>/register` for a single interaction, or import `@citolab/prose-qti/components/register` to register every interaction at once.

## Inserting an interaction

In the editor UI, use the **Interactions** toolbar button to open the insert menu. Only interactions whose schema nodes are registered in your editor extension are listed. Each item is disabled when the cursor is in a position where that node cannot be inserted.

Interactions can also be inserted programmatically using the command functions exported from each package:

```ts
import { insertChoiceInteraction } from '@citolab/prose-qti/components/choice';
import { insertHottextInteraction } from '@citolab/prose-qti/components/hottext';
import { insertTextEntryInteraction } from '@citolab/prose-qti/components/text-entry';
import { insertInlineChoiceInteraction } from '@citolab/prose-qti/components/inline-choice';
import { insertSelectPointInteraction } from '@citolab/prose-qti/components/select-point';

// Call as a command:
insertChoiceInteraction(view.state, view.dispatch, view);
```

## Attributes panel

When the cursor is inside any interaction node, the **Attributes** panel shows the schema-defined attributes for that node and its ancestors. Changes are applied live to the editor document.

Attributes vary by interaction type but commonly include:

| Attribute | Description |
|---|---|
| `responseIdentifier` | Links the interaction to a response declaration |
| `maxChoices` | Maximum number of selectable options (choice interaction) |
| `expectedLength` | Character limit hint (text entry / extended text) |
| `caseSensitive` | Whether the response match is case-sensitive (text entry) |

The [framework guides](/docs/frameworks/react/) and [framework guides](/docs/frameworks/angular/) show complete integration examples.
