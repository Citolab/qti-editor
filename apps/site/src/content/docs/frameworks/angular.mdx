This guide shows two Angular integration approaches:

- **Pure ProseMirror** - Minimal dependencies with full control over editor behavior
- **ProseKit** - Pre-built UI components and convenience wrappers for faster development

Both approaches use the same QTI interaction packages. We've provided a ready-made [Angular integration example](https://github.com/Citolab/qti-editor-angular) to serve as inspiration.

If you want Angular to own the toolbar, inspector, and side-panel UI instead of using the registry web components, see [Angular Custom UI](/docs/frameworks/angular/custom-ui/).

:::note
QTI Editor is a composable kit — it provides the QTI building blocks that slot into an existing ProseMirror instance. It does not ship integration packages for specific frameworks or third-party libraries, and dedicated integration support is outside its scope. The guides in this documentation illustrate one way to wire things together; how you integrate the kit into your application is up to you. 
:::

## Approach 1: Pure ProseMirror

### 1. Install packages

```bash
pnpm add prosemirror-state prosemirror-view prosemirror-model prosemirror-commands \
  @qti-components/theme \
  @citolab/prose-qti \
  lit
```

### 2. Create ProseMirror editor service

```typescript
// src/app/services/qti-editor.service.ts
import { Injectable } from '@angular/core';
import { EditorState } from 'prosemirror-state';
import { EditorView } from 'prosemirror-view';
import { Schema } from 'prosemirror-model';
import { 
  listInteractionDescriptors, 
  listInteractionPluginFactories 
} from '@citolab/prose-qti/core/interactions/composer';

@Injectable({ providedIn: 'root' })
export class QtiEditorService {
  createEditor(element: HTMLElement): EditorView {
    const descriptors = listInteractionDescriptors();
    
    // Build schema from descriptors
    const nodes = {};
    const seenSpecs = new Set<string>();
    
    for (const descriptor of descriptors) {
      for (const { name, spec } of descriptor.nodeSpecs) {
        if (!seenSpecs.has(name)) {
          nodes[name] = spec;
          seenSpecs.add(name);
        }
      }
    }

    const schema = new Schema({
      nodes: {
        doc: { content: 'block+' },
        text: {},
        paragraph: { content: 'inline*', group: 'block' },
        ...nodes
      },
      marks: {},
    });

    // Get plugins from interaction packages
    const plugins = listInteractionPluginFactories().map(factory => factory());

    const state = EditorState.create({ schema, plugins });
    return new EditorView(element, { state });
  }
}
```

## Approach 2: ProseKit (Recommended for Rapid Development)

### 1. Install packages

```bash
pnpm add prosekit \
  @qti-components/theme \
  @citolab/prose-qti \
  @citolab/prose-extensions \
  lit
```

### Notes on ProseKit Integration

- The GitHub registry in this repository is the source for copied UI components; do not treat the demo app shell components as a runtime import path.
- ProseKit integration is optional — interaction packages export pure ProseMirror plugins.
- The intended public model is: `prosekit` (optional) + `@citolab/prose-qti` + `@citolab/prose-extensions` + copied registry UI.

### Install the Lit UI through the registry

```bash
pnpm dlx shadcn@latest add @prosekit/lit-example-toolbar
pnpm dlx shadcn@latest add citolab/qti-editor/qti-attributes-panel
pnpm dlx shadcn@latest add citolab/qti-editor/qti-interaction-insert-menu
```

## ProseKit-Specific Setup (Only for Approach 2)

If you're using ProseKit (Approach 2), wire the integration surfaces from the packages directly — no copy-paste required.

### Editor events and interactions

```ts
// src/app/app.ts
import '@citolab/prose-qti/components/register';
import { defineQtiInteractionsExtension } from '@citolab/prose-qti/integration/interactions/prosekit';
import { qtiEditorEventsExtension } from '@citolab/prose-qti/integration/events';
import { blockSelectExtension } from '@citolab/prose-extensions/block-select';
import { nodeAttrsSyncExtension } from '@citolab/prose-extensions/node-attrs-sync';
import { defineSemanticPasteExtension } from '@citolab/prose-extensions/paste-semantic-html';
import {
  defineLocalStorageDocPersistenceExtension,
  readPersistedStateFromLocalStorage,
} from '@citolab/prose-extensions/local-storage-doc-persistence-extension';
import { defineBasicExtension } from 'prosekit/basic';
import { createEditor, union } from 'prosekit/core';
```

`listInteractionPluginFactories()` is called internally by `defineQtiInteractionsExtension`. It registers runtime ProseMirror plugins for authoring behavior such as correct-response updates or wrapping selected text as hottext — do not omit it.

## Mount the editor from an Angular host component

Angular owns the layout and lifecycle. The copied Lit elements receive the editor instance via refs.

```ts
// src/app/app.ts
import {
  CUSTOM_ELEMENTS_SCHEMA,
  ChangeDetectorRef,
  Component,
  ElementRef,
  NgZone,
  ViewChild,
  inject,
} from '@angular/core';
import { FormsModule } from '@angular/forms';
import { createEditor, union, type Editor } from 'prosekit/core';
import { defineBasicExtension } from 'prosekit/basic';
import { blockSelectExtension } from '@citolab/prose-extensions/block-select';
import { nodeAttrsSyncExtension } from '@citolab/prose-extensions/node-attrs-sync';
import { defineSemanticPasteExtension } from '@citolab/prose-extensions/paste-semantic-html';
import {
  defineLocalStorageDocPersistenceExtension,
  readPersistedStateFromLocalStorage,
} from '@citolab/prose-extensions/local-storage-doc-persistence-extension';
import { defineQtiInteractionsExtension } from '@citolab/prose-qti/integration/interactions/prosekit';
import { qtiEditorEventsExtension } from '@citolab/prose-qti/integration/events';
import '@citolab/prose-qti/components/register';

import '../components/button';
import '../components/image-upload-popover';
import '../components/slash-menu';
import '../components/toolbar';
import '../components/composer';
import '../components/composer-metadata-form';
import '../components/attributes-panel';
import '../components/interaction-insert-menu';
import '../components/convert-menu';

const STORAGE_KEY = 'qti-editor-angular:prosemirror-doc:v1';

@Component({
  selector: 'app-root',
  imports: [FormsModule],
  templateUrl: './app.html',
  styleUrl: './app.css',
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
})
export class App {
  @ViewChild('toolbar', { static: true })
  private readonly toolbarRef?: ElementRef<HTMLElement & { editor: Editor | null }>;

  @ViewChild('slashMenu', { static: true })
  private readonly slashMenuRef?: ElementRef<HTMLElement & { editor: Editor | null }>;

  @ViewChild('insertMenu', { static: true })
  private readonly insertMenuRef?: ElementRef<HTMLElement & { editor: Editor | null }>;

  @ViewChild('convertMenu', { static: true })
  private readonly convertMenuRef?: ElementRef<HTMLElement & { editor: Editor | null }>;

  @ViewChild('attributesPanel', { static: true })
  private readonly attributesPanelRef?: ElementRef<HTMLElement & { editor: Editor | null }>;

  @ViewChild('composer', { static: true })
  private readonly composerRef?: ElementRef<HTMLElement & {
    editor: Editor | null;
    liveComposeEnabled: boolean;
  }>;

  @ViewChild('mount')
  set mountRef(ref: ElementRef<HTMLDivElement> | undefined) {
    const el = ref?.nativeElement ?? null;
    if (!el) return;
    queueMicrotask(() => this.ngZone.runOutsideAngular(() => this.mountCurrentEditor(el)));
  }

  protected latestHtml = '';

  private readonly cdr = inject(ChangeDetectorRef);
  private readonly ngZone = inject(NgZone);
  private readonly eventTarget = new EventTarget();
  private readonly editor = createEditor({
    extension: union(
      defineBasicExtension(),
      defineQtiInteractionsExtension(),
      defineSemanticPasteExtension(),
      defineLocalStorageDocPersistenceExtension({ storageKey: STORAGE_KEY }),
      blockSelectExtension,
      nodeAttrsSyncExtension,
      qtiEditorEventsExtension({ eventTarget: this.eventTarget }),
    ),
    defaultContent: readPersistedStateFromLocalStorage(STORAGE_KEY).doc,
  });

  constructor() {
    this.eventTarget.addEventListener('qti:content:change', event => {
      this.latestHtml = (event as CustomEvent<{ html: string }>).detail.html;
      this.cdr.detectChanges();
    });
  }

  private mountCurrentEditor(el: HTMLElement): void {
    el.innerHTML = '';
    this.editor.mount(el);

    queueMicrotask(() => {
      this.toolbarRef!.nativeElement.editor = this.editor;
      this.slashMenuRef!.nativeElement.editor = this.editor;
      this.insertMenuRef!.nativeElement.editor = this.editor;
      this.convertMenuRef!.nativeElement.editor = this.editor;
      this.attributesPanelRef!.nativeElement.editor = this.editor;
      this.composerRef!.nativeElement.editor = this.editor;
      this.composerRef!.nativeElement.liveComposeEnabled = true;
    });
  }
}
```

```html

    <div class="editor-toolbar">
      <div class="editor-toolbar-row">
        <div class="editor-qti-tools">
```

## Angular-specific notes

- Add `CUSTOM_ELEMENTS_SCHEMA` to the host component.
- Import the copied registry files for side effects so the custom elements register.
- Assign `editor`, `editorView`, and `eventTarget` via refs, not HTML attributes.
- `defineQtiInteractionsExtension` and `qtiEditorEventsExtension` come from `@citolab/prose-qti`; no local copies needed.
