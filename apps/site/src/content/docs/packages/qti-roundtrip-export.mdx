`@qti-editor/qti-package-builder` exports the editor's ProseMirror document to a **standard QTI 3.0 package**.

## Scope

The package builder:

- writes valid QTI 3.0 package files
- folds editor authoring state (correct responses, scores) into standard `qti-response-declaration` / `qti-response-processing`
- emits interactions with **no editor-specific attributes** (no `data-*` mirrors, no marker attributes)

The output is interchange-friendly standard QTI 3.0. On re-import, `@qti-editor/qti3-item-import` reconstructs authoring state from the native declarations (see [Itembody-only QTI Subformat](./itembody-subformat)).

## Main exports

```ts
import {
  createQtiPackageFromNode,
  createQtiPackageFromProseMirrorJson,
  createQtiPackageFromItems,
} from '@qti-editor/qti-package-builder';
```

## Typical usage

Export directly from the current editor document:

```ts
import { createQtiPackageFromNode } from '@qti-editor/qti-package-builder';

const blob = await createQtiPackageFromNode(editor.view.state.doc, {
  identifier: 'item-1',
  title: 'Sample Item',
  lang: 'en',
  packageIdentifier: 'sample-package',
  testTitle: 'Sample Test',
});
```

You can then download the returned `Blob` or upload it to your own backend.

## Package contents

The generated ZIP contains:

- `imsmanifest.xml`
- `assessment-test.xml`
- one or more item XML files under `items/`
- referenced image assets, when present

Multiple items are supported. The exporter uses the same item-fragment logic as the QTI composer and can package several assessment items into one assessment test.

## Metadata context

The context object supports both item-level and package-level metadata:

- `identifier`
- `title`
- `lang`
- `packageIdentifier`
- `testIdentifier`
- `testTitle`
- `sectionIdentifier`
- `sectionTitle`

## Roundtrip model

Authoring state is preserved through **standard QTI 3.0**, not through editor-specific attributes:

- correct responses → `qti-correct-response` values in `qti-response-declaration`
- scores → `qti-response-processing`

On import, `@qti-editor/qti3-item-import` (`roundtripQtiItem`) hoists these back onto each interaction as canonical, unprefixed attributes. Because the export is standard QTI, the same importer can also ingest third-party QTI 3.0 items — though editor-only hints with no standard source (text-entry `case-sensitive`, select-point `area-mappings`) are only present when the source item carried them.
