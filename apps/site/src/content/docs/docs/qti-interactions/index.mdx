QTI Editor ships its interactions as focused packages. Each `@qti-editor/interaction-*` package adds the schema, commands, serialization, web components, and any runtime ProseMirror plugins needed for one QTI interaction type.

These packages are closely coupled to our other open source library, [`qti-components`](https://github.com/Citolab/qti-components). In practice, QTI Editor is responsible for authoring behavior and document structure, while `qti-components` supplies the underlying QTI web components that render and encapsulate the interaction behavior itself. That split keeps the integration framework agnostic while letting each interaction package stay small and explicit.

When you install an interaction package, its matching `@qti-components/*` dependency is pulled in automatically. You typically install only the interactions your product needs.

## Available interactions

| Interaction | What it does | Package |
|---|---|---|
| [Choice Interaction](/docs/qti-interactions/choice/) | Single- or multiple-choice questions with a prompt and selectable choices | `@qti-editor/interaction-choice` |
| [Hottext Interaction](/docs/qti-interactions/hottext/) | Marking words or phrases inside a text passage as selectable response options | `@qti-editor/interaction-hottext` |
| [Inline Choice Interaction](/docs/qti-interactions/inline-choice/) | Dropdown choices embedded directly in a sentence | `@qti-editor/interaction-inline-choice` |
| [Text Entry Interaction](/docs/qti-interactions/text-entry/) | Short free-text answers entered inline | `@qti-editor/interaction-text-entry` |
| [Extended Text Interaction](/docs/qti-interactions/extended-text/) | Longer free-text answers in a multi-line response area | `@qti-editor/interaction-extended-text` |
| [Match Interaction](/docs/qti-interactions/match/) | Matching items across two sets | `@qti-editor/interaction-match` |
| [Order Interaction](/docs/qti-interactions/order/) | Ordering a list of choices into the correct sequence | `@qti-editor/interaction-order` |
| [Select Point Interaction](/docs/qti-interactions/select-point/) | Clicking a coordinate or hotspot on an image | `@qti-editor/interaction-select-point` |

## Installing interactions

Install only the packages you need, for example:

```bash
pnpm add \
  @qti-editor/interaction-choice \
  @qti-editor/interaction-hottext \
  @qti-editor/interaction-inline-choice \
  @qti-editor/interaction-text-entry
```

Each interaction package exports [ProseMirror](https://prosemirror.net/) commands and schema pieces. Interactions that need extra runtime behavior, such as click-to-mark-correct-response or wrapping selected text as hottext, now ship their required ProseMirror plugins with the package as well.

If you assemble your editor through the descriptor-driven registry (`@qti-editor/core` plus `@qti-editor/prosekit-integration`), those runtime plugins are installed automatically.

If you assemble a plain ProseMirror editor yourself, use `listInteractionDescriptors()` for node specs and commands and `listInteractionPluginFactories()` for the runtime plugins to install on the editor state.

The [framework guides](/docs/frameworks/react/) and [framework guides](/docs/frameworks/angular/) show complete integration examples.
