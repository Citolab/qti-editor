QTI Editor was built for teams that need to create and maintain QTI content inside their own products.

In practice, that usually means helping test makers author assessment items without forcing them to work directly in raw QTI XML. Instead of hand-editing markup, they get a WYSIWYG editing experience that can turn structured input into valid QTI. They can write and edit content directly in the editor, or start from content pasted in from other documents and continue refining it there.

The editor is meant to be embedded wherever QTI editing capability is needed, such as item banks, assessment platforms, and other authoring workflows. Because the UI is built from web components, the integration model is framework agnostic: you can use it from plain web apps or wire it into frameworks such as React and Angular.

Use these guides to choose an integration path, install the packages you need, and add QTI editing to your application.

QTI Editor's interaction packages are built on top of [`@qti-components`](https://github.com/Citolab/qti-components) — Citolab's library of QTI-compliant web components that handle interaction rendering and runtime behavior. Each `@qti-editor/interaction-*` package pulls in its matching `@qti-components` dependency automatically.

QTI Editor is part of a broader set of QTI tooling maintained by Citolab. For an overview of the full toolkit, visit [qti.citolab.nl](https://qti.citolab.nl/).
