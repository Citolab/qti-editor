# Building Your Own QTI Editor

The QTI Editor is a modular, open-source editor for authoring
[QTI 3.0](https://www.imsglobal.org/spec/qti/v3p0) assessment items.
It is built on [ProseKit](https://prosekit.dev) (a toolkit on top of ProseMirror)
and uses [Lit](https://lit.dev) web components for the UI layer.

You can use it as a complete, ready-made editor or pick individual packages to
compose your own.

## Architecture overview

The editor is split into three layers:

- **ProseMirror extensions** — node specs for QTI interactions, plus
  utility plugins (block select, paste normalization, attribute syncing).
- **ProseKit integration** — wraps ProseMirror extensions for use with
  ProseKit's `createEditor` / `union` API, plus event helpers and item context.
- **UI components** — Lit web components for the toolbar, slash menu,
  attributes panel, interaction insert menu, and more. Distributed as a
  [shadcn-style registry](https://ui.shadcn.com/docs/registry) so you can copy
  them into your project and modify freely.

## Packages

| Package | Purpose |
|---|---|
| `@qti-editor/core` | QTI XML composer — turns editor state into valid QTI 3.0 XML |
| `@qti-editor/prosemirror` | Pure ProseMirror plugins: block select, virtual cursor, node-attrs sync, semantic paste, local-storage persistence |
| `@qti-editor/prosekit-integration` | ProseKit wrappers for events, item context, and interaction helpers |
| `@qti-editor/interaction-choice` | Choice interaction (multiple choice / multiple response) |
| `@qti-editor/interaction-text-entry` | Inline text entry (fill-in-the-blank) |
| `@qti-editor/interaction-extended-text` | Extended text (free-form text area) |
| `@qti-editor/interaction-match` | Match interaction (table-based matching) |
| `@qti-editor/interaction-select-point` | Select-point interaction (click on an image) |
| `@qti-editor/interaction-inline-choice` | Inline choice (dropdown within text) |
| `@qti-editor/interaction-shared` | Shared node specs and web components used by all interaction packages |
| `@qti-editor/ui` | Lit UI components — toolbar, slash menu, attributes panel, composer, etc. |

## Create a new project

Start with a Vite project using the Lit template:

```bash
npm create vite@latest my-qti-editor -- --template lit-ts
cd my-qti-editor
```

Install the core dependencies:

```bash
npm install lit prosekit @lit/context
```

Install the QTI Editor packages you need:

```bash
npm install @qti-editor/core @qti-editor/prosemirror @qti-editor/prosekit-integration
npm install @qti-editor/interaction-choice @qti-editor/interaction-shared
```

Add more interaction packages as needed (`@qti-editor/interaction-text-entry`,
`@qti-editor/interaction-extended-text`, `@qti-editor/interaction-match`, etc.).

## Copy UI components from the registry

The UI components are distributed as a shadcn-style registry. Install them with
the shadcn CLI:

```bash
npx shadcn@latest add https://qti-editor.citolab.nl/r/lit-editor-toolbar.json
npx shadcn@latest add https://qti-editor.citolab.nl/r/qti-attributes-panel.json
npx shadcn@latest add https://qti-editor.citolab.nl/r/qti-interaction-insert-menu.json
```

These components are copied into your project as source files. You own them and
can modify them however you like.
